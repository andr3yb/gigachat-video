📋 ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ) НА КУРСОВОЙ ПРОЕКТ
AI Video Factory on Kandinsky 5.0
Веб-сервис для генерации видео по тексту и изображениям через GigaChat API, FastAPI и Docker
 
1. ОСНОВНАЯ ИНФОРМАЦИЯ
Параметр	Значение
Команда	1–3 человека (каждый участвует во всём)
Срок выполнения	2–3 недели
Платформа	Готовая виртуалка Debian (предоставляется преподавателем)
Технологии	GigaChat API (Kandinsky 5.0), FastAPI, PostgreSQL, Celery, Docker, React/Next.js
 
2. СУТЬ ПРОЕКТА (ЧТО ДЕЛАЕМ)
Создать веб-сервис, где пользователь загружает изображение и пишет текстовый промпт, а нейросеть Kandinsky 5.0 от Сбера превращает это в 5-секундное видео с анимацией. Всё упаковано в Docker и поднимается одной командой на Debian.
Kandinsky 5.0 — это открытая модель, доступная через GigaChat API, отлично понимает русский язык и работает в РФ без ограничений .
 
3. ЧТО УМЕЕТ KANDINSKY 5.0 (ДЛЯ ПОНИМАНИЯ СТУДЕНТАМИ)
Характеристика	Значение
Что делает	Превращает статичное изображение в анимированное видео
Длительность	5 секунд (фиксировано)
Качество	512P или 1024P
Стоимость	$0.20 (512P) / $0.60 (1024P) за видео
Вход	URL изображения + текстовый промпт
Выход	MP4-видео с публичной ссылкой

Студенты не платят реальные деньги — для учебных целей используются тестовые ключи или небольшие гранты.
 
4. РОЛИ ТЕХНОЛОГИЙ В ПРОЕКТЕ
🎨 Kandinsky 5.0 (GigaChat API) — сердце проекта
Задача: Генерировать видео по изображению и тексту.
Что делает:
•	Принимает URL картинки (студенты загружают через интерфейс)
•	Принимает текстовый промпт ("кот в шляпе танцует")
•	Возвращает ссылку на готовое 5-секундное MP4-видео
Технически: HTTP-запросы к fal.ai/kandinsky5-pro/image-to-video, обработка асинхронного ответа, сохранение видео.
 
🧩 Веб-интерфейс (React/Next.js)
Задача: Дать пользователю удобный интерфейс.
Что делает:
•	Форма загрузки изображения (drag-and-drop)
•	Поле для ввода промпта (с подсказками)
•	Выбор качества (512P / 1024P)
•	Галерея сгенерированных видео
•	История запросов пользователя
 
⚡ FastAPI — бэкенд-оркестратор
Задача: Управлять запросами и очередью.
Что делает:
•	Принимает файлы изображений и сохраняет их в облачное хранилище (или локально с URL)
•	Сохраняет промпты и метаданные в PostgreSQL
•	Отправляет задачи в очередь Celery
•	Получает готовые видео и отдаёт ссылки фронтенду
•	Ведёт историю генераций
Эндпоинты:
•	POST /api/generate — запустить генерацию
•	GET /api/tasks/{id} — узнать статус задачи
•	GET /api/videos — список всех сгенерированных видео
•	GET /api/videos/{id} — скачать конкретное видео
 
🐳 Docker + Celery — очередь задач
Задача: Не потерять запросы и обрабатывать их асинхронно.
Что делает:
•	Ставит запросы в очередь (чтобы сервис не падал при пиковых нагрузках)
•	Параллельно обрабатывает несколько генераций
•	Пробует повторить запрос при ошибке (retry-логика)
Пример retry-логики (из документации):
python
def generate_with_retry(image_url, prompt, max_retries=3):
    for attempt in range(max_retries):
        result = call_kandinsky_api(image_url, prompt)
        if result["success"]:
            return result
        time.sleep(2 ** attempt)  # exponential backoff
    return {"error": "Max retries exceeded"}
 
🗄️ PostgreSQL — база данных
Хранит:
•	Пользователей (если есть авторизация)
•	Промпты и настройки генерации
•	Ссылки на готовые видео
•	Статистику (сколько времени заняла генерация)
 
🐙 Docker Compose — оркестратор
Поднимает одной командой:
•	postgres — база данных
•	redis — брокер сообщений для Celery
•	backend — FastAPI приложение
•	celery_worker — воркеры для обработки задач
•	frontend — React/Next.js интерфейс
•	nginx — reverse proxy (опционально)
 
5. АРХИТЕКТУРА СИСТЕМЫ (ВИЗУАЛЬНО)

6. ТРЕБОВАНИЯ К ПРОЕКТУ (ЧТО ДОЛЖНО РАБОТАТЬ)
Модуль 1: Загрузка и промпт
•	Пользователь может загрузить изображение (JPG, PNG, WebP)
•	Есть поле для ввода текстового промпта
•	Есть выбор качества (512P / 1024P)
•	Кнопка "Сгенерировать видео"
Модуль 2: Обработка и очередь
•	После нажатия показывается статус "В очереди..." → "Генерируется..." → "Готово"
•	Если API временно недоступен, задача не теряется (повторные попытки)
•	Несколько пользователей не мешают друг другу (очередь)
Модуль 3: Галерея и результаты
•	Список всех сгенерированных видео с превью
•	Возможность скачать видео
•	Возможность удалить видео
•	Показывается промпт, под которым видео создано
Модуль 4: Docker-контейнеризация
•	Все сервисы поднимаются одной командой docker-compose up
•	На хосте (ваша Debian) не нужно ставить ничего кроме Docker
•	Переменные окружения (API-ключи) хранятся в .env
